# Contributing to MFlow Benchmarks

Thank you for your interest in contributing to MFlow Benchmarks!

## How to Contribute

### Reporting Issues

- Use GitHub Issues to report bugs or suggest improvements
- Include detailed steps to reproduce any issues
- Provide system information and error logs when applicable

### Submitting Changes

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/your-feature`)
3. Make your changes
4. Run any applicable tests
5. Commit with clear messages (`git commit -m "Add: description"`)
6. Push to your fork (`git push origin feature/your-feature`)
7. Open a Pull Request

### Code Style

- Python: Follow PEP 8 guidelines
- Shell: Use `set -euo pipefail` for error handling
- Documentation: Use clear, concise English

### Adding New Benchmarks

When adding a new benchmark:

1. Create a new directory under `benchmarks/`
2. Include a `README.md` with:
   - Dataset description
   - Evaluation methodology
   - Results table
   - Reproduction instructions
3. Include all necessary scripts
4. Provide sample results for verification

## Questions?

Open an issue for any questions about contributing.
